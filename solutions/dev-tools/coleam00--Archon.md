## What It Does
Archon is an open-source AI coding workflow engine that lets teams define software development processes as YAML workflows, making AI-assisted coding deterministic and repeatable. It orchestrates multi-agent pipelines for issue fixing, feature development, code review, and refactoring — running each workflow in an isolated git worktree to prevent conflicts.

## Use Cases
- Automated issue triage and fix: classify → investigate → implement → test → open PR, all hands-free
- Feature scaffolding: convert a ticket/idea into production-ready code with validation loops
- Parallel AI development: run 10+ concurrent feature branches without git conflicts
- Code review automation: multi-agent PR analysis with synthesized feedback
- Safe refactoring: execute large refactors with type-checking guards at each step
- CI/CD integration: trigger workflows via GitHub webhooks on issue assignment

## Who Uses It in Production
- Growing community of AI-forward dev teams (17.4K stars, April 2026)
- Indie developers and small teams replacing manual code-review pipelines
- Compatible with Claude Code, Codex, OpenClaw, and OpenCode providers

## Tech Stack
- Runtime: Bun (TypeScript)
- Workflow engine: YAML-defined pipelines with agent step types
- Git: isolated worktrees per workflow run
- Storage: SQLite (default) or PostgreSQL for multi-node
- Web UI: built-in Archon dashboard (`archon serve`)
- Integrations: GitHub, GitLab, Telegram, Slack, Discord webhooks

## How to Deploy
1. Install CLI: `curl -fsSL https://archon.coleam.dev/install.sh | bash` or `brew install coleam00/tap/archon`
2. Interactive setup: `archon init` (configures credentials, project root, LLM provider)
3. Run a workflow: `archon run fix-issue --issue 123`
4. Web dashboard: `archon serve` → http://localhost:4200
5. Docker/VPS: `docker compose up -d` using provided compose file (PostgreSQL backend recommended)
6. Connect GitHub webhooks: `archon integrations github setup` for auto-trigger on issue assignment

## Deploy Requirements
- **Min RAM:** 512 MB (2 GB+ for parallel workflows)
- **Min CPU:** 1 vCPU (2+ for parallel worktree execution)
- **Storage:** 1 GB + workspace per concurrent worktree (~200 MB each)
- **External deps:** LLM provider API key (Claude, OpenAI, etc.); Git installed
- **Estimated cost:** Free self-hosted on any VPS; LLM API costs vary by workflow complexity

## Limitations & Gotchas
- YAML workflow DSL has a learning curve; complex custom workflows take time to author
- LLM-generated code quality varies — always require human review before merging PRs
- Bun runtime required (not Node.js) — may require pipeline changes in some CI systems
- PostgreSQL recommended for production; SQLite mode loses durability guarantees
- No built-in code sandboxing — generated code runs with local user privileges
